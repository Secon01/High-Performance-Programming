With private(a), each thread gets its own uninitialized copy of a, so a += b uses an indeterminate value; therefore a is unpredictable, while b = 20 is consistent because it is shared. 
The output order is also non-deterministic due to parallel execution. After changing to firstprivate(a), each thread’s a is initialized to 100, so all threads compute a = 120 deterministically. 
The lastprivate clause copies the value from the logically last iteration back to the original variable after the parallel region.
