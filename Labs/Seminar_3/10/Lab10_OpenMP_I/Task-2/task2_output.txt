Why `long int`?
Because the loop runs billions of iterations. A 32-bit `int` can store at most 2,147,483,647, so it would overflow. `long int` safely handles larger values.

Does wall time change with more threads?
No significant change. Each thread performs the same full workload, so the total work is not reduced. Increasing threads does not speed up the program.

What happens when threads > cores?
Threads compete for CPU time. The OS performs context switching (oversubscription), which can slightly increase wall time.

What does `omp_get_wtime()` measure?
It measures wall-clock time (real elapsed execution time).
