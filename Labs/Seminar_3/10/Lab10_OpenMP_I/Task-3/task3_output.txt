OpenMP assigns each thread a unique ID using omp_get_thread_num(), ranging from 0 to n−1,
where n is the total number of threads. The function omp_get_num_threads() returns the number
of threads currently active inside the parallel region, while omp_get_max_threads() returns
the maximum number of threads that OpenMP would use for a parallel region based on the current runtime settings. 
Running the program with different values of OMP_NUM_THREADS confirms that the thread IDs range correctly from 0 
to n−1 and that the total number of active threads matches the requested number.
